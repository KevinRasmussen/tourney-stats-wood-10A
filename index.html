📋 Production Operational Procedures

1. Persistence Rules

  • Data syncs via Cloud RTDB. All rosters, opponents, and logged stints persist through page hard-refreshes or device restarts.
  • **Switch Safety:** Changing active pitchers when uncommitted counts are greater than 0 triggers a confirmation warning to insulate logs against user data-loss.

2. Sorting Metrics Architecture

  • Rosters are organized dynamically based on current total tournament pitch capacity in **descending order**. Tie-breaking drops back to standard alphanumeric filters.
